A tiny static HTML page + an example workflow to show how Actions runs on push.

What this demo does

This repository contains a single index.html file. The included GitHub Actions workflow (see below) runs on every push and performs a quick check to ensure index.html exists and prints its size. It's intentionally minimal so you can see the flow of data and logs in Actions.

How to use

  1. Create a new GitHub repo (or use an existing one).
  2. Copy this file and commit it as index.html at the repository root.
  3. Create the workflow file at .github/workflows/ci.yml (example below) and push.
  4. Open the Actions tab on GitHub to watch the run logs.

Example workflow (save as .github/workflows/ci.yml) 

name: simple-check

on:
  push:
    branches: [ main, master ]

jobs:
  check-index:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repo
        uses: actions/checkout@v4

      - name: Verify index.html exists and show size
        run: |
          echo "Listing root files:"
          ls -la
          if [ -f index.html ]; then
            echo "index.html found — size (bytes):"
            wc -c index.html
          else
            echo "index.html NOT FOUND — failing the job"
            exit 1
          fi

What you'll see in Actions

When the workflow runs you'll see each step in the Actions UI. The Verify index.html step prints the listing and size or fails the job if the file is missing — useful as a smoke check for static sites.
